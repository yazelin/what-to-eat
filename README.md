## ✨ 功能

- **三個分類轉盤**：早餐、午晚餐、飲料，一鍵切換。
- **✏️ 編輯菜單**：三個分類都能新增／刪除／改名／換圖，改完即時反映到轉盤；用 **localStorage 保存**，重整仍在，並可一鍵「還原預設」。新增餐點從內建圖示挑選。
- **SVG 動態轉盤**：扇形、食物 icon、名稱皆由 JS 產生；文字與圖案**永遠正立**（轉完不歪）。
- **隨機抽選**：至少轉 5 圈、ease-out 減速，指針精準對齊中選格。
- **中獎連鎖動畫**：中央笑臉大笑 → 噴出愛心／閃光 → 全螢幕彩帶灑落 → 結果卡彈出。
- **即時合成音效**：背景音樂、轉動「答答答」聲（與減速完全同步）、中獎琶音、點擊音，全部用 Web Audio API 合成，**不需要任何音檔**。
- **輕量素材**：52 張圖以 WebP（保留透明）交付，總大小僅 ~4.9MB。
- **RWD**：桌機雙欄（轉盤＋結果卡）、手機單欄。
- **FB 粉絲團按鈕**：右上角導流。

## 🚀 執行方式

**方法一：直接開啟**

雙擊 `index.html` 即可。

**方法二：本機伺服器（建議，SVG 圖示最穩定）**

雙擊 `start.bat`，會自動用 Python 起一個本機伺服器並開啟瀏覽器；沒有 Python 則退而直接開檔。

```bash
# 或手動
python -m http.server 8000
# 開 http://localhost:8000
```

預設餐點寫在 `script.js` 的 `DEFAULT_MEALS`，使用者的編輯結果存在 localStorage（`whatToEat.meals.v1`）。圖片使用相對路徑，新增餐點只要一筆 `{ name, icon }`。

## 🛠️ 技術重點

### 轉盤文字圖案「恆正立」
整個轉盤旋轉時，若文字圖案一起轉，停在隨機角度就會全歪。解法是給每個 icon／文字包一層 `<g>` 做**反向旋轉**：螢幕上的傾角 = `轉盤角度 − 最終角度`，當轉盤轉到最終角度時傾角歸零 → 完全正立，且仍落在正確扇形內。

### 轉動音效與減速同步
不是用近似曲線硬湊，而是用轉盤實際的 `cubic-bezier(0.16, 1, 0.3, 1)` 反推每次「格子邊界經過指針」的時間點——每轉過一格響一聲，開頭密集、結尾稀疏，和眼睛看到的減速一格不差。

### 純合成音訊
單一 `AudioContext`，SFX 與 BGM 走各自的 gain node。背景音樂用三角波旋律 + 正弦波低音循環排程；因瀏覽器自動播放規範，於使用者首次點擊時啟動。

### 編輯菜單 + 保存
`DEFAULT_MEALS` 為預設來源，實際使用的是可變的 `meals`；編輯後寫入 localStorage 並經 `loadMeals()` 結構驗證，隱私模式以 try/catch 退回記憶體。切換分類與編輯共用 `refreshWheel()` 重建轉盤，新增／刪除後格數、字級、正立與抽選皆自動適應（每分類最少保留 2 項）。

## 🎨 去背 skill

`assets/飲料/` 的飲料圖以「**邊緣氾濫填色**」去背：從四邊 flood fill，只清掉與邊緣相連的近背景像素，保留主體內部同色細節（吸管、高光、白色貼紙描邊）。此方法已封裝成可重用的 Claude Code skill：

```bash
python .claude/skills/remove-bg/remove_bg.py "assets/飲料" --backup --check check.png
```

## 🎨 AI 美術素材生成流程

本專案的美術素材採用 AI 輔助生成，再經過整理、命名、去背與前端整合。目標不是單純產生漂亮圖片，而是建立一組風格一致、能實際用在網頁 App 裡的 UI 素材。

### 1. 先確定美術方向

一開始先定義整體風格：

* 可愛便當遊戲風
* 日系療癒感
* 柔和粉彩色
* 圓潤線條
* 乾淨、不要太花
* 適合繁體中文介面
* 適合放在餐點轉盤、結果卡、按鈕與背景中使用

主視覺概念是「像小遊戲一樣的餐點轉盤」，所以素材不是寫實照片，而是可愛、貼紙感、icon 化的插畫。

### 2. 先產生整體 UI 概念圖

在正式生單張素材前，先請 AI 產生一張完整的 UI mockup，用來確認整體方向。

範例 prompt：

```text
請產生一張可愛便當遊戲風的網頁 App 概念圖。

主題是「今天要吃什麼？」餐點轉盤工具。
畫面需要有：
- 標題「今天要吃什麼？」
- 早餐、午餐、晚餐切換 Tab
- 中央大轉盤
- 轉盤上有餐點名稱與可愛食物 icon
- 上方有固定指針
- 下方有「幫我決定！」按鈕
- 右側有結果卡，例如「今天早餐吃：蛋餅 + 豆漿」
- 下方有編輯菜單區

風格：
可愛、療癒、日系便當風、柔和粉彩色、圓潤線條、乾淨、像小遊戲 UI。
背景使用奶油色，可有淡淡食物 doodle 和格紋桌布。
```

這一步的重點是確認「看起來像什麼」，不是直接拿這張圖當網頁背景。

### 3. 拆分素材類型

確認風格後，把素材拆成幾類：

```text
assets/
├─ 主頁/      # 背景、指針、裝飾 icon
├─ 早餐/      # 早餐餐點 icon
├─ 午晚餐/    # 午餐與晚餐餐點 icon
└─ 飲料/      # 飲料 icon
```

這樣前端可以依分類載入素材，也比較容易維護。

### 4. 食物 icon 生成方式

食物 icon 採用「一張圖一個品項」的方式生成，方便放進轉盤格子裡。

範例 prompt：

```text
請產生一張單獨的食物 icon。

主題：蛋餅

風格：
可愛、療癒、日系便當遊戲風、柔和粉彩色、圓潤線條、乾淨、貼紙感插畫。
食物可以有可愛笑臉與腮紅，但不要太複雜。
構圖置中，適合放在網頁 App 的轉盤中使用。
背景請保持乾淨，不要加文字，不要加多餘場景。
```

後續其他餐點沿用同樣句型，只替換主題，例如：

```text
主題：牛肉麵
主題：咖哩飯
主題：壽司
主題：珍珠奶茶
```

這樣可以讓同一批圖維持一致的視覺語言。

### 5. 用參考圖維持風格一致

當第一批圖片的風格確認後，後續生成新素材時，會把已經滿意的圖作為風格參考，並在 prompt 裡強調：

```text
請參考前面圖片的風格，只參考畫風，不要複製內容。
請維持同樣的：
- 柔和粉彩色
- 圓潤線條
- 可愛笑臉
- 乾淨構圖
- 貼紙感
- 適合 64px 到 128px icon 使用
```

這一步很重要，否則每次生成的圖可能會變成不同畫風。

### 6. 主頁裝飾素材

主頁素材包含：

* 指針
* 星星
* 飯糰角色
* 彩帶
* 愛心
* 閃光
* 豆漿
* 吐司
* 咖啡
* 背景

範例 prompt：

```text
請產生一張可愛便當遊戲風的裝飾 icon。

主題：紅粉色轉盤指針

風格：
可愛、療癒、日系便當遊戲風、柔和粉彩色、圓潤線條、乾淨、貼紙感。
用途是放在餐點轉盤上方當固定指針。
構圖置中，不要文字，不要背景場景。
```

背景圖則使用不同 prompt：

```text
請產生一張 1920x1080 的網頁背景圖。

風格：
奶油色、可愛、療癒、日系便當風。
背景要非常淡，不可以搶主畫面。
可以有食物 doodle、愛心、星星、杯子、吐司、豆漿盒。
下方可以有淡淡格紋桌布。
中間保留大量空白，方便放 UI。
不要文字。
```

### 7. 飲料 icon 改成白底

食物與主頁裝飾原本偏向透明背景或去背用途，後來飲料素材改成白底，方便後續去背與視覺統一。

飲料生成 prompt 範例：

```text
請產生一張單獨的飲料 icon。

主題：珍珠奶茶

風格：
可愛、療癒、日系便當遊戲風、柔和粉彩色、圓潤線條、乾淨、貼紙感插畫。
飲料杯要有可愛笑臉與腮紅。
底圖請使用純白色背景。
不要文字，不要多餘裝飾。
構圖置中，適合放在網頁 App 的轉盤中使用。
```

飲料列表包含：

* 珍珠奶茶
* 紅茶
* 綠茶
* 奶茶
* 美式咖啡
* 拿鐵
* 豆漿
* 冬瓜茶
* 檸檬紅茶
* 水果茶

### 8. 檔名整理

圖片生成後，全部改成中文檔名，讓素材和餐點名稱可以直接對應。

例如：

```text
assets/早餐/蛋餅.png
assets/早餐/蔥抓餅.png
assets/午晚餐/牛肉麵.png
assets/午晚餐/咖哩飯.png
assets/飲料/珍珠奶茶.png
assets/飲料/綠茶.png
assets/主頁/指針.png
assets/主頁/背景.png
```

前端資料也直接使用相同名稱：

```js
{ name: "蛋餅", icon: "assets/早餐/蛋餅.png" }
{ name: "牛肉麵", icon: "assets/午晚餐/牛肉麵.png" }
{ name: "珍珠奶茶", icon: "assets/飲料/珍珠奶茶.png" }
```

這樣未來新增素材時，只要檔名和資料名稱一致，就很好維護。

### 9. 去背處理

飲料圖因為是白底產生，後續使用自製去背流程處理。

做法不是單純把所有白色刪掉，而是使用「邊緣 flood fill」：

1. 從圖片四邊開始找背景色
2. 只移除和邊緣相連的白色背景
3. 保留主體內部的白色高光、吸管、杯蓋、描邊
4. 輸出成可放在轉盤上的透明 PNG

這個流程已整理成 Claude Code skill：

```bash
python .claude/skills/remove-bg/remove_bg.py "assets/飲料" --backup --check check.png
```

### 10. 給 AI Agent 實作網頁時的重點

在請 AI Agent 寫網頁時，要特別強調：

```text
不要把整個 UI 做成一張圖片。
轉盤、Tab、按鈕、結果卡都必須是 HTML/CSS/JS 可互動元件。
圖片素材只用在 icon、指針、裝飾與背景。
請使用相對路徑，不要使用 D:\ 開頭的絕對路徑。
```

這樣做的好處是：

* 轉盤可以真的旋轉
* 餐點可以由 JS 資料控制
* 圖片可以重複利用
* RWD 比較容易調整
* 未來新增餐點或分類更簡單

### 11. 素材生成心得

這次流程的重點是「先定風格，再批次生成，再整理命名，最後交給前端整合」。

實務上建議順序是：

1. 先做一張整體 UI 概念圖
2. 挑出喜歡的美術方向
3. 生成第一批 10 張 icon
4. 用第一批滿意圖當風格參考
5. 繼續生成早餐、午晚餐、飲料
6. 產生主頁裝飾素材與背景
7. 統一中文命名
8. 必要時去背
9. 再交給 AI Agent 實作前端

不要一開始就要求 AI Agent「照圖做完整網站」，而是要把素材和功能分開處理。這樣比較容易得到可維護、可互動、也真的能玩的網頁 App。
